The analog readout should have a hour, minute, and second hand. **Implementation:** The implementation of the digital face was pretty straightforward. The digital display time is formatted using the different display integer variables (hours, minutes, seconds). One problem I ran into was the issue of excessive blinking on drawing/redrawing each of the values. So I included checks for when to update the seconds, minutes, and hours. Instead of using the fillScreen() function to essentially erase old values, I used the fillRoundRect() function erase old values. I should also note I manually drew in the colons for precision of division between hours, minutes, and seconds. ``` void draw_digital() { if (update_title){ tft.fillScreen(TFT_BLUE); tft.drawString("DIGITAL",0,0,1); //draw colons tft.fillCircle(15, 12, 1, TFT_WHITE); tft.fillCircle(15, 14, 1, TFT_WHITE); tft.fillCircle(35, 12, 1, TFT_WHITE); tft.fillCircle(35, 14, 1, TFT_WHITE); update_title = false; } if (update_hours) { tft.fillRoundRect(0, 10, 10, 10, 0, TFT_BLUE); tft.drawNumber(display_hours, 0, 10, 1); update_hours = false; } if (update_minutes) { tft.fillRoundRect(20, 10, 10, 10, 0, TFT_BLUE); tft.drawNumber(display_minutes, 20, 10, 1); update_minutes = false; } if (update_seconds) { tft.fillRoundRect(40, 10, 15, 15, 0, TFT_BLUE); tft.drawNumber(display_seconds, 40, 10, 1); update_seconds = false; } } ``` The implementation of the analog face was more nuanced, though. Since it's forbidden to use analog clock display libraries, getting the coordinates for the hours/minute/second hands required basic trigonometry. Below are the calculations and variable assignments I used for the analog watch face. Since the clock display had to be smooth (and not flash from being redrawn every time), I used boolean variables to determine whether minutes/hours hands had to be redrawn. The seconds hand essentially sweeps over and erases the minute and hours hands when its drawn and redrawn. To account for this, I added a check for when the hours and minutes values are near the seconds value. Then, the program can update the hands and the hands don't get erased. I should also note, the hour hand had to be adjusted according to 2 factors. First, I used the modulo function to handle military time differences on analog. Second, I added the offset of the amount of minutes passed to mirror an analog clock in real life. ![Basic trig for analog clock.](https://raw.githubusercontent.com/trudypainter/608/master/exercise1/tpainter_watch/trig.jpeg =100x20) ~~~ void draw_analog() { // title screen if (update_title){ tft.fillScreen(TFT_BLUE); tft.drawString("ANALOG", 0, 0, 1); // draw clock circle tft.drawCircle(center, center, 30, TFT_WHITE); } // use an adjusted hour value to account for military time and minutes passed float adjusted_hour = fmod(display_hours, 12.0) + display_minutes/60.0; if (update_seconds) { // draw seconds tft.drawLine(center, center, sx, sy, TFT_BLUE); // use past var sy = (second_hand * cos(pi-(2*pi)/60 * display_seconds)) + center; sx = (second_hand * sin(pi-(2*pi)/60 * display_seconds)) + center; tft.drawLine(center, center, sx, sy, TFT_WHITE); update_seconds = false; // to account for the seconds hand erasing the other clock hands // (since everything is drawn and redrawn) // a buffer is added because of the pixelated lines update_minutes = ((display_seconds - 10 < display_minutes && display_minutes < display_seconds) || update_title); update_hours = ((display_seconds - 10 < adjusted_hour*5 && adjusted_hour*5 < display_seconds) || update_title); } if (update_minutes) { // draw minutes tft.drawLine(center, center, mx, my, TFT_BLUE); // use past var my = (minute_hand * cos(pi-(2*pi)/60 * display_minutes)) + center; mx = (minute_hand * sin(pi-(2*pi)/60 * display_minutes)) + center; tft.drawLine(center, center, mx, my, TFT_WHITE); update_minutes = false; } if (update_hours) { // draw hours tft.drawLine(center, center, hx, hy, TFT_BLUE); // use past var hy = (hour_hand * cos(pi-(2*pi)/12 * adjusted_hour)) + center; hx = (hour_hand * sin(pi-(2*pi)/12 * adjusted_hour)) + center; tft.drawLine(center, center, hx, hy, TFT_WHITE); update_hours = false; } update_title = false; } ~~~ ## ðŸ” Switching Between Faces **Requirement:** * The user should be able swap between the two faces using a button connected to input **pin 5.**. Switching between the two faces should be based on pushing-and-then-releasing the button. One push-release sets it to analog. Another to digital. Another to analog. And so on. It cannot be pushed=analog/unpushed=digital for the faces. The button functionality must be sufficiently responsive that its usage isn't impaired at any time during operation (i.e. you should avoid **blocking** code with the exception of the GET request). **Implementation:** I used 4 different states outlined below. The initial state is the digital face. The user can alternate between faces through press-release actions. ![State diagram for button pushes.](https://raw.githubusercontent.com/trudypainter/608/master/exercise1/tpainter_watch/states.jpeg =5x) ## â˜ï¸ Server Time Retrieval and Smooth Updating **Requirement:** * In order to get absolute time, your device must make periodic HTTP GET requests to the url here: http://iesc-s3.mit.edu/esp32test/currenttime which will return the current Boston time in a format like the following: `2018-02-11 18:37:29.776430` * The device needs to only report the time (don't worry about temperature or anything else in the video), but it can't just repeatedly grab it off the server. Instead *at most* it can grab the time once every one minute, and in the time in between in must update based off of its own internal timer. **Implementation:** I refactored some of the code from lab1b for GET requests and connecting to my wifi. To manage smooth transitions between the universal time and the local time (calculated from the arduino timer), I used "universal time" variables (hours, minutes, seconds) and "display time" variables (display_hours, display_minutes, display_seconds). ~~~ // TIMER VARS uint32_t timer; char universal_time[1000]; char digital_display[1000]; uint32_t time_since_update; uint8_t hours; uint8_t minutes; uint8_t seconds; uint8_t display_hours; uint8_t display_minutes; uint8_t display_seconds; uint8_t minute_passed; ~~~ My code makes calls to the 6.08 currenttime resource during these times: 1. On startup, in the setup() function. This is to initialize the universal time variables. 2. Every 60 seconds, it makes another call to currenttime resource and (1) updates the universal times and (2) resets the last time since update in the arduino local time calculation ~~~ void get_universal_time() { sprintf(request_buffer, "GET /esp32test/currenttime HTTP/1.1\r\n"); strcat(request_buffer, "Host: iesc-s3.mit.edu\r\n"); //add more to the end strcat(request_buffer, "\r\n"); //add blank line! do_http_GET("iesc-s3.mit.edu", request_buffer, response_buffer, OUT_BUFFER_SIZE, RESPONSE_TIMEOUT, true); Serial.println(response_buffer); //print to serial monitor //set universal time vars memcpy(universal_time, &response_buffer[11], 8); //to format hh:mm:ss hours = atoi(universal_time); memmove(universal_time, universal_time+3, strlen (universal_time)); //move the universal time pointer to get ints minutes = atoi(universal_time); memmove(universal_time, universal_time+3, strlen (universal_time)); //move the universal time pointer to get ints seconds = atoi(universal_time); //initialize the display vars display_seconds = seconds; display_minutes = minutes; display_hours = hours; minute_passed = 0; //start timer to update time_since_update = millis(); } void update_time(){ //timeout check to update the current time if ((millis() - time_since_update) > 60000) { Serial.println("** GETTING NEW TIME UPDATE **"); get_universal_time(); } //set the display time variables display_seconds = seconds + ((millis() - time_since_update)/1000) - minute_passed; if (display_seconds > 59) { display_seconds = 0; minute_passed = 60; display_minutes++; } if (display_minutes > 59) { display_minutes = 0; display_hours++; } if (display_hours > 12) { display_hours = 1; } } ~~~ # ðŸ’» Code Appendix All of my code for this project is below. ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ #include //Connect to WiFi Network #include // Graphics and font library for ST7735 driver chip #include //Used in support of TFT Display #include //used for some string handling and processing.
