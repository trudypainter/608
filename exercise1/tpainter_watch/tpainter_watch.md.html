The analog readout should have a hour, minute, and second hand. **Implementation:** The implementation of the digital face was straightforward. The digital display time is formatted using the different display integer variables (hours, minutes, seconds). ``` sprintf(digital_display, "%d : %d : %d", display_hours, display_minutes, display_seconds); ``` The implementation of the analog face was more nuanced, though. Since it's forbidden to use analog clock display libraries, getting the coordinates for the hours/minute/second hands required basic trigonometry. Below are the calculations and variable assignments I used for the analog watch face. ![Basic trig for analog clock.](https://raw.githubusercontent.com/trudypainter/608/master/exercise1/tpainter_watch/trig.jpeg =100x20) ~~~ void draw_analog() { // draw seconds y = (second_hand * cos(pi-(2*pi)/60 * display_seconds)) + center; x = (second_hand * sin(pi-(2*pi)/60 * display_seconds)) + center; tft.drawLine(center, center, x, y, TFT_WHITE); // draw minutes y = (minute_hand * cos(pi-(2*pi)/60 * display_minutes)) + center; x = (minute_hand * sin(pi-(2*pi)/60 * display_minutes)) + center; tft.drawLine(center, center, x, y, TFT_WHITE); // draw hours y = (hour_hand * cos(pi-(2*pi)/12 * display_hours)) + center; x = (hour_hand * sin(pi-(2*pi)/12 * display_hours)) + center; tft.drawLine(center, center, x, y, TFT_WHITE); // draw clock circle tft.drawCircle(center, center, 30, TFT_WHITE); } ~~~ ## üîÅ Switching Between Faces **Requirement:** * The user should be able swap between the two faces using a button connected to input **pin 5.**. Switching between the two faces should be based on pushing-and-then-releasing the button. One push-release sets it to analog. Another to digital. Another to analog. And so on. It cannot be pushed=analog/unpushed=digital for the faces. The button functionality must be sufficiently responsive that its usage isn't impaired at any time during operation (i.e. you should avoid **blocking** code with the exception of the GET request). **Implementation:** I used 4 different states outlined below. The initial state is the digital face. The user can alternate between faces through press-release actions. ![State diagram for button pushes.](https://raw.githubusercontent.com/trudypainter/608/master/exercise1/tpainter_watch/states.jpeg =5x) ## ‚òÅÔ∏è Server Time Retrieval and Smooth Updating **Requirement:** * In order to get absolute time, your device must make periodic HTTP GET requests to the url here: http://iesc-s3.mit.edu/esp32test/currenttime which will return the current Boston time in a format like the following: `2018-02-11 18:37:29.776430` * The device needs to only report the time (don't worry about temperature or anything else in the video), but it can't just repeatedly grab it off the server. Instead *at most* it can grab the time once every one minute, and in the time in between in must update based off of its own internal timer. **Implementation:** I refactored some of the code from lab1b for GET requests and connecting to my wifi. To manage smooth transitions between the universal time and the local time (calculated from the arduino timer), I used "universal time" variables (hours, minutes, seconds) and "display time" variables (display_hours, display_minutes, display_seconds). ~~~ // TIMER VARS uint32_t timer; char universal_time[1000]; char digital_display[1000]; uint32_t time_since_update; uint8_t hours; uint8_t minutes; uint8_t seconds; uint8_t display_hours; uint8_t display_minutes; uint8_t display_seconds; uint8_t minute_passed; ~~~ My code makes calls to the 6.08 currenttime resource during these times: 1. On startup, in the setup() function. This is to initialize the universal time variables. 2. Every 60 seconds, it makes another call to currenttime resource and (1) updates the universal times and (2) resets the last time since update in the arduino local time calculation ~~~ void get_universal_time() { sprintf(request_buffer, "GET /esp32test/currenttime HTTP/1.1\r\n"); strcat(request_buffer, "Host: iesc-s3.mit.edu\r\n"); //add more to the end strcat(request_buffer, "\r\n"); //add blank line! do_http_GET("iesc-s3.mit.edu", request_buffer, response_buffer, OUT_BUFFER_SIZE, RESPONSE_TIMEOUT, true); Serial.println(response_buffer); //print to serial monitor //set universal time vars memcpy(universal_time, &response_buffer[11], 8); //to format hh:mm:ss hours = atoi(universal_time); memmove(universal_time, universal_time+3, strlen (universal_time)); //move the universal time pointer to get ints minutes = atoi(universal_time); memmove(universal_time, universal_time+3, strlen (universal_time)); //move the universal time pointer to get ints seconds = atoi(universal_time); //initialize the display vars display_seconds = seconds; display_minutes = minutes; display_hours = hours; minute_passed = 0; //start timer to update time_since_update = millis(); } void update_time(){ //timeout check to update the current time if ((millis() - time_since_update) > 60000) { Serial.println("** GETTING NEW TIME UPDATE **"); get_universal_time(); } //set the display time variables display_seconds = seconds + ((millis() - time_since_update)/1000) - minute_passed; if (display_seconds > 59) { display_seconds = 0; minute_passed = 60; display_minutes++; } if (display_minutes > 59) { display_minutes = 0; display_hours++; } if (display_hours > 12) { display_hours = 1; } } ~~~ # üíª Code Appendix All of my code for this project was in the following file: ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ #include //Connect to WiFi Network #include // Graphics and font library for ST7735 driver chip #include //Used in support of TFT Display #include //used for some string handling and processing.
